Modals

Use .c-dialog component classes for styling modal dialogs.

Dialog

The default component class is used to style a simple dialog box. Add absolute positioning properties in order to logically associate the dialog with its opener.

Open .c-dialog Dialog

Centered Dialog

A dialog with additional positioning CSS to center it on the page. The dialog retains its centered position until the window shrinks to a small size, at which breakpoints activate to position the dialog to the top and/or left of the viewport. Extend the centered dialog component with @media queries that reposition wider/longer dialogs used for your particular application.

Open .c-dialog .c-dialog--center Dialog

Centered Dialog with Default Backdrop Layout

Surround modal dialogs with a backdrop to give them visual distinction from page content. The backdrop layout also adds scrolling behavior so that dialog content can be accessed in a small browser window. Note: implementations are expected to toggle "overflow: hidden" on the <html> element in order to prevent scrolling of background content (and potential double scrollbars for both the content and the backdrop). The backdrop may add a listener, as this demo does, that closes the dialog when a user clicks outside the dialog.

Centered Dialog with Lightbox Backdrop Layout

The backdrop may be modified to support a darker lightbox treatment.

Dialog with Transparent Backdrop Layout

There may be cases where interaction outside of a dialog needs to be handled or prevented. The transparent backdrop element supports this use case. The transparent backdrop does not add it's own scrolling behavior.
